# Step6 编码->代码验收

## 任务描述

将`tasks/`中的任务列表，按照依赖顺序逐一完成编码和代码评审，**循环**执行每个任务，直到完成所有任务。

## 执行过程

### Phase1 [工程化编码]

- 任务描述：通知研发工程师完成对应任务的[工程化编码]
- 执行角色：研发工程师 `code_developer`

### Phase2 [工程化代码审查]

- 任务描述：通知代码审核员完成对应任务的[工程化代码审查]
- 执行角色：代码验收工程师 `code_reviewer`
- 注意：区分`单元测试`与`测试用例`

### Phase3 AI审查

- 任务描述：查看`reviews/`目录下对应任务的`review-task-{任务编号}.md`文档，决策审查是否通过

#### 可选的审查决策

**通过**：进入下一个任务的执行，更新`state.json`记录任务执行状态

**不通过**：
- 在`reviews/review-report.md`文档中记录审查、循环修复过程
- 修改`tasks/`对应的task文件，把`Must Fix`项加入到task文件中
- 返回Phase1，通知研发工程师完整重做任务
- 每个任务的修复循环最多2轮

## state.json 中的 tasks 状态

```json
{
    "tasks": [
        {"id": "Task-001", "name": "...", "status": "已完成"},
        {"id": "Task-002", "name": "...", "status": "进行中"},
        {"id": "Task-003", "name": "...", "status": "待开发"}
    ]
}
```

状态值：`待开发` → `进行中` → `待审查` → `修复中-第N轮` → `已完成`
